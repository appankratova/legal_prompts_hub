  # Интеграция с государственными реестрами РФ для M&A: техно-гайд

Для расширения возможностей промптов — используйте данные открытых государственных реестров. Это повысит точность due diligence, disclosure letter, судебного анализа.

---

## 1. ЕГРЮЛ (ед. гос. реестр юр. лиц)

**Где брать данные:**
- https://egrul.nalog.ru
- https://api-fns.ru/ (API)

**Параметры для запросов:**
- ИНН, ОГРН компании
- полное/краткое наименование

**Какие данные парсить:**
- юридический адрес
- список учредителей/бенефициаров
- виды деятельности (ОКВЭДы)
- истории изменений, сведения о банкротстве/ликвидации
- действующие лица и доверенности

**Формат результата:**  
JSON/CSV

**Пример вызова (API fns):**
    GET https://api-fns.ru/api/search?q=ИНН

---

## 2. Росреестр (реестр недвижимости)

**Где брать данные:**
- https://rosreestr.gov.ru
- https://esia.gosuslugi.ru (через сервисы Госуслуги)

**Параметры:**
- кадастровый номер
- адрес объекта

**Что парсить:**
- права собственности и обременения
- переходы прав, списки заявленных притязаний
- связанные лица (владельцы, арендаторы)

---

## 3. КАД Арбитр (суды)

**Где брать данные:**
- https://kad.arbitr.ru
- сторонние парсеры (libs: kad-arbitr, arbparser, etc)

**Параметры:**
- ИНН, ОГРН компании
- ФИО, роль в деле

**Что парсить:**
- количество и статус дел (истец/ответчик)
- тематика споров (M&A, корпоративные конфликты, IP, налоги)
- результат рассмотрения (выигрыш, проигрыш, мировое)
- суммы требований и принятые меры

---

## 4. СПАРК/Контур.Фокус (платные B2B)

**Что дают:**
- Глубокая связность компаний (аффилированность, "цепочки влияния")
- Финансовые показатели, риск-скоринг

**Параметры:**  
- ИНН/ОГРН, или название компании

**!!! Для платных сервисов — нужны акк и/или ключ API**

---

## Как автоматизировать интеграцию: чек-лист

1. Получить/собрать список ИНН или реквизитов нужных компаний/объектов.
2. Для каждого сервиса автоматизировать парсинг (например, через requests, aiohttp, curl или единую библиотеку API).
3. Собрать данные в единый JSON/CSV и использовать их в input промпта.

- Построить карту аффилированных лиц и рисков (через все реестры)
